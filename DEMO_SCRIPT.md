## 🎬 Scene 1. 시스템 아키텍처 (2분)

### 📍 화면
`http://localhost:3005/architecture` 접속

### 📝 하실 말

> "시작하기 전에 전체 시스템 구조를 간단히 보여드리겠습니다."

**[화면 상단 ← 사용자 질문 박스를 가리키며]**

> "사용자가 질문을 하면, 가장 먼저 **에이전트 라우터**가 동작합니다.
> 질문 의도를 분석해서 상담·분석·추천 중 최적의 AI 에이전트로 자동 라우팅합니다.
> 여기서 **Intent 분류, Slot Filling, 자율 추론**이 수행됩니다."

**[화면 중간 ← 3개 에이전트 영역을 가리키며]**

> "에이전트는 3가지입니다.
> - **상담 에이전트**: 기본 정보 안내, 용어 설명
> - **분석 에이전트**: 수익률 분석, 상품 비교, 차트 생성
> - **추천 에이전트**: 맞춤 ETF 추천, 포트폴리오 제안
>
> 각 에이전트는 자기 역할에 맞는 MCP 도구만 사용합니다."

**[화면 하단 ← MCP Server Layer를 가리키며]**

> "아래쪽이 **MCP Server Layer**입니다. Model Context Protocol 기반 8개 도구가 있습니다.
> ETF 검색, 수익률 분석, 비교분석, 문서 RAG, 뉴스검색, ETF 추천, 경쟁사 대안 매칭까지.
> AI가 스스로 필요한 도구를 선택해서 호출합니다.
>
> 이 전체 아키텍처가 RFP에서 요구하신 **멀티에이전트 + MCP + RAG** 구조입니다.
>
> 이제 실제로 작동하는 것을 보여드리겠습니다."

## 🎬 Scene 2. 가드레일 — 매매 권유 거절 (2분) 🛡️

### 📍 화면
`http://localhost:3005` 메인 챗봇 화면으로 이동

### 📝 하실 말 (입력 전)

> "가장 먼저 **금융사에서 가장 민감한 부분**부터 보여드리겠습니다.
> 일반 고객이 가장 많이 하는 질문 — '이거 사야 해 말아야 해?'를 쳐보겠습니다."

### ⌨️ 입력
```
KODEX 200 사야 해 말아야 해?
```

### ⏳ 대기 (10~15초)
AI 응답을 기다리는 동안 이렇게 말하세요:

> "지금 AI 처리 과정이 실시간으로 표시되고 있습니다. 에이전트 라우팅, MCP 도구 호출, 가드레일 검증이 순차적으로 진행 중입니다."

### ✅ 기대 결과 (실제 테스트에서 확인됨)

1. **AI 처리 과정 로그** (접으면 숨김 가능):
   - 🤖 에이전트 라우팅 → 상담 에이전트
   - 🔍 MCP Server [ETF상품] 검색
   - ✅ KODEX 200 상세 정보 조회 완료
   - 📊 MCP Server [수익률] 수익률 데이터 + 가격 히스토리 조회
   - 📰 MCP Server [뉴스검색]
   - 🛡️ Guardrails [컴플라이언스 검증] 응답 필터링 시작
   - ✅ 금칙어 검사 통과 (0건)
   - 🛡️ Guardrails [투자 권유 검증] 매수/매도 권유 여부 확인
   - ✅ 투자 권유 표현 없음 확인
   - ✅ 면책조항 포함 확인

2. **AI 답변 내용**:
   - **거절 문구**: "투자 판단은 투자자 본인의 결정 영역이므로, 구체적인 매수/매도 권유는 드리기 어렵습니다."
   - **객관적 데이터 테이블**: 1년 수익률 +133.14%, 3년 +161.53%, AUM 16,239억원, 보수 0.15%, MDD 등
   - **가격 추이 차트**: KODEX 200 최근 1년 라인 차트 자동 생성
   - **면책조항**: "투자 판단은 투자자 본인의 책임입니다"

### 📝 하실 말 (결과 보며)

> "보시면 AI가 **즉시 거절**합니다. '매수/매도 권유는 드리기 어렵습니다.'
>
> 하지만 여기서 끝이 아닙니다. **거절 후에도 고객이 이탈하지 않도록** 객관적인 데이터를 테이블과 차트로 제공합니다.
>
> *(AI 처리 과정을 펼쳐서 보여주며)*
>
> 여기 보시면 Guardrails가 **4단계 자동 검증**을 수행했습니다:
> 1. 금칙어 검사 — '반드시 사세요', '원금 보장' 같은 11개 금칙어 패턴 검사
> 2. 투자 권유 검증 — '사세요', '파세요' 같은 직접적 권유 표현 없는지 2차 검증
> 3. 할루시네이션 체크 — 도구에서 조회하지 않은 수치를 만들어냈는지 확인
> 4. 면책조항 확인 — 없으면 자동 삽입
>
> 이 로그는 **준법감시 감사(Audit) 자료**로 활용 가능합니다. 모든 대화에 자동으로 남습니다."

### 💡 임원별 어필 포인트
| 대상 | 핵심 메시지 |
|---|---|
| 준법감시 | "4단계 자동 필터 + 감사 로그, 수동 검토 부담 제로" |
| 디지털 | "거절해도 데이터 제공 → 고객 이탈률 최소화" |
| 상품기획 | "거절 응답에서도 KODEX 상품 데이터가 자연스럽게 노출" |

## 🎬 Scene 4. ETF 상세 조회 + 차트 자동 생성 (2분) 📊

> "이번에는 정상적인 ETF 질문을 해보겠습니다. 반도체 ETF 수익률을 물어볼게요."

### ⌨️ 입력
```
KODEX 반도체 ETF 수익률이 어떻게 되나요?
```

### ⏳ 대기 (10~15초)

> "지금 분석 에이전트가 활성화되어 반도체 관련 ETF를 검색하고, 수익률과 가격 데이터를 조회하고 있습니다."

### ✅ 기대 결과 (실제 테스트에서 확인됨)

1. **9개 반도체 관련 KODEX ETF** 자동 검색
2. **기간별 수익률 데이터** (1M, 3M, 6M, 1Y, 3Y)
3. **가격 추이 차트 자동 생성** — KODEX 반도체MV, 반도체레버리지 등 종목별 라인 차트

### 📝 하실 말 (결과 보며)

> "하나의 질문에 AI가 자동으로:
> 1. 반도체 관련 KODEX ETF 9개를 검색하고
> 2. 주요 종목의 수익률 데이터를 조회하고
> 3. **가격 추이 차트를 자동 생성**했습니다
>
> RFP에서 요구하신 '**수치 데이터의 경우 표(Table) 또는 차트(Chart) 라이브러리와 연동 가능한 JSON 구조화 응답**'이 바로 이것입니다.
> 차트는 Recharts 라이브러리로 실시간 렌더링됩니다."

## 🎬 Scene 5. 슬롯 필링 — 대화형 AI (3분) 🎯

### 📝 하실 말 (입력 전)

> "이번에는 일부러 **애매하게** 물어보겠습니다. 일반 챗봇과 대화형 AI의 차이를 보여드리겠습니다."

### ⌨️ 입력 (1차)
```
요즘 반도체 관련 ETF 뭐가 좋아?
```

### ⏳ 대기 (5~10초)

### ✅ 기대 결과 — 1차 (실제 테스트에서 확인됨)

AI가 **바로 답변하지 않고 되묻습니다**:
- 1️⃣ **국내 반도체** ETF를 찾으시나요, **미국 반도체** ETF를 찾으시나요?
- 2️⃣ **수익률** 기준으로 찾으시나요, **수수료(보수)** 기준으로 찾으시나요?

### 📝 하실 말 (1차 결과 보며)

> "보셨죠? 일반 챗봇이라면 아무 반도체 ETF나 나열했을 겁니다.
> 하지만 저희 AI는 **되묻습니다.** '국내인지 미국인지?', '수익률 기준인지 수수료 기준인지?'
>
> 이것이 RFP에 명시된 **Slot Filling**입니다.
> AI가 정확한 답변을 위해 필요한 정보를 **대화를 통해 확보**합니다."

### ⌨️ 입력 (2차 — 사용자 답변)
```
국내 반도체, 수익률 기준으로 알려줘
```

### ⏳ 대기 (10~15초)

### ✅ 기대 결과 — 2차

- 국내 반도체 KODEX ETF를 수익률 기준으로 정렬
- 수익률 테이블 + 차트 생성
- 상품별 비교 분석

### 📝 하실 말 (2차 결과 보며)

> "사용자가 '국내 반도체, 수익률 기준'이라고 답하니까
> 이제 정확한 조건으로 도구를 호출해서 **맞춤 결과**를 제공합니다.
>
> **2번의 대화로 정확한 답변에 도달**합니다.
> 이것은 단순 키워드 매칭이 아니라, **대화 맥락을 이해하는 AI**라는 증거입니다.
>
> 콜센터 상담에서도 이 패턴이 적용되면 고객 니즈를 정확히 파악해서 **1회 상담 해결률**이 올라갑니다."

## 🎬 Scene 6. 경쟁사 대안 비교 — AI 세일즈맨 (2분) 🔄⭐

### 📝 하실 말 (입력 전)

### ⌨️ 입력
```
TIGER 미국나스닥100 대신 KODEX로 갈아탈 수 있어?
```

> "지금 AI가 자동으로 `find_kodex_alternative`라는 도구를 호출했습니다.
> TIGER 미국나스닥100에 대응하는 KODEX 상품을 찾고 있습니다."

### ✅ 기대 결과 (실제 테스트에서 확인됨)

1. **KODEX 대안 3종 자동 매칭**:
   - KODEX 미국나스닥100
   - KODEX 미국나스닥100(H) — 환헷지 버전
   - KODEX 미국나스닥100데일리커버드콜OTM
2. **레이더 차트** — 수익률·안정성·수수료·규모·성장성 5축 비교
3. **개별 가격 차트** — 각 종목 1년 추이
4. **Next Best Action 버튼 3개**:
   - "보수/성과 직접 비교가 필요하시면 어떻게 하시겠어요?"
   - "환헷지(H) 상품이란 무엇인가요?"
   - "나스닥100 ETF의 최근 시장 동향이 궁금하시다면?"

### 📝 하실 말 (결과 보며)

> "TIGER라는 경쟁사 상품명을 말했는데, AI가 자동으로 KODEX 대안을 3개 찾아습니다.
>
> 레이더 차트를 보시면 — 수익률, 안정성, 수수료, 규모, 성장성 5개 축으로 비교합니다.
> **보수가 KODEX가 더 낮다면, 차트에서 자연스럽게 부각**됩니다.
>
> 이건 사실상 **'KODEX를 24시간 영업하는 AI 세일즈맨'**입니다.
>
> 고객이 TIGER, ACE, RISE, SOL 어떤 경쟁사를 검색하든
> 항상 KODEX 대안이 자동 노출됩니다. 별도 마케팅 비용 없이요.
>
> 그리고 하단에 **Next Best Action** — 후속 질문 버튼이 자동 생성됩니다.
> 고객이 자연스럽게 다음 탐색으로 이어지면서 **앱 체류시간이 늘어나는 구조**입니다."

### 💡 핵심 어필 포인트
